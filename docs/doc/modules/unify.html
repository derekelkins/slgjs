    • Defined in unify.ts:346

    Match x against y except only require that for any objects in x, their keys are a subset of the corresponding keys of y. For example, looseMatch({},{foo:1}) succeeds, but looseMatch({foo:1},{}) does not.

    Parameters

    • x: any

      A JsonTerm to loosely unify against. This may contain Variable.

    • y: any

      A Json, potentially with additional keys, to loosely match against. This contains no Variables even bound ones.

    • sub: Substitution<any>

      Bindings for the Variables in x.

    Returns Substitution<JsonTerm> | null

    null if the match fails, otherwise sub extended with bindings such that x grounds to a JsonTerm that is identical to y except that it may have fewer keys in any objects.

looseUnifyJson

  • looseUnifyJson(x: any, y: any, sub: Substitution<any>): Substitution<JsonTerm> | null
    • Defined in unify.ts:399

    Unify x against y except only require that for any objects in x, their keys are a subset of the corresponding keys of y. This makes looseUnifyJson asymmetric unlike unifyJson. For example, looseUnify({},{foo:1}) succeeds, but looseUnify({foo:1},{}) does not. This does not perform an occurs check.

    Parameters

    • x: any

      A JsonTerm to loosely unify against.

    • y: any

      A JsonTerm potentially with additional keys to loosely unify against.

    • sub: Substitution<any>

      Bindings for the Variables in x and y.

    Returns Substitution<JsonTerm> | null

    null if unification fails, otherwise sub extended with bindings such that x grounds to a JsonTerm that is identical to the grounding of y except that it may have fewer keys in any objects.
